## Identifying a possible project

I have had two ideas regarding a potential project.

My first idea is to create a food recipes app in either VueJS, or React.
This app would allow users to create an account, in order to share thier food recipes with other users.

There would be a component where users can input a name for their recipe, a description of the recipe, ingredients for the recipe, and the steps to create the recipe.

There would be another component which would function as a sort of board of recipes, where users could go to view all of the recipes that had been added to the website, and see who they were added by.

Another functionality could be that users could add recipes that they like to a list of favourites, so they can find them again easily.

Users could possibly also leave comments on recipes.


The second idea I had was to create some sort of web app in two difference frameworks (VueJS and React), and compare them in regards, to complexity, learning curve, data binding, state management etc.

I think it would be interesting to compare the current most popular framework (React) to the framework that I anticipate to be the most popular (VueJS) when building the exact same web app.
